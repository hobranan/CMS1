# Use Case UC-16: Generate Conference Schedule

**Goal in Context**: An authorized admin/editor manually generates a conference schedule draft from accepted papers, reviews conflicts, and publishes the schedule.  
**Scope**: Conference Management System (CMS)  
**Level**: User Goal  
**Primary Actor**: User  
**Secondary Actors**: CMS services, data store  
**Trigger**: The primary actor initiates this use case.

## Success End Condition
* Published schedule is available for retrieval.

## Failed End Condition
* The use case does not complete, and the system returns an explicit, actionable failure state.

## Preconditions
* User is authenticated as admin/editor.
* Conference has accepted papers.
* Required room and slot parameters are configured.

## Main Success Scenario
1. Primary actor performs the primary action for this use case.
2. System returns expected success outcome.

## Extensions
* **1a**: No accepted papers: block with setup error.
* **2a**: Missing parameters: block with setup error.
* **3a**: Save failure: return system error and persist nothing.
* **4a**: Blocking conflicts: prevent publish until resolved.
* **5a**: Cancel publish: leave draft unpublished.
