# Use Case UC-17: Edit Conference Schedule

**Goal in Context**: Authorized admins/editors modify an existing published schedule, save valid edits, and keep published visibility with updated last-edited metadata.  
**Scope**: Conference Management System (CMS)  
**Level**: User Goal  
**Primary Actor**: User  
**Secondary Actors**: CMS services, data store  
**Trigger**: The primary actor initiates this use case.

## Success End Condition
* Valid edits are saved; schedule remains published; latest edit timestamp is visible.

## Failed End Condition
* The use case does not complete, and the system returns an explicit, actionable failure state.

## Preconditions
* User is authenticated as admin/editor.
* Published schedule exists for conference.
* Schedule is not policy-locked.

## Main Success Scenario
1. Primary actor performs the primary action for this use case.
2. System returns expected success outcome.

## Extensions
* **1a**: Cancel before save: no changes persisted.
* **4a**: Invalid references/conflicts: save rejected with explanatory feedback.
* **4b**: Policy lock active: save blocked.
* **1b**: Stale version: save rejected with reload instruction.
* **1c**: DB failure: no persisted partial change.
