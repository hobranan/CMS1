# Use Case UC-14: Record Paper Decision

**Goal in Context**: TBD  
**Scope**: Conference Management System (CMS)  
**Level**: User Goal  
**Primary Actor**: User  
**Secondary Actors**: CMS services, data store  
**Trigger**: The primary actor initiates this use case.

## Success End Condition
* Decision is stored and paper status is updated to `accepted` or `rejected`.

## Failed End Condition
* The use case does not complete, and the system returns an explicit, actionable failure state.

## Preconditions
* Editor is authenticated.
* Paper exists and belongs to the editor.
* Decision period is open.
* At least one completed review exists unless an explicit override is used.

## Main Success Scenario
1. Primary actor performs the primary action for this use case.
2. System returns expected success outcome.

## Extensions
* **1a**: No completed reviews and no override: block decision with explanation.
* **2a**: Paper already decided: return conflict and keep existing status.
* **3a**: Decision period closed: block decision.
* **4a**: Cancel before confirmation: no mutation.
* **5a**: Save failure: return error, keep status unchanged.
* **6a**: Notification failure: decision remains committed with failed notification status.
