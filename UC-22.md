# Use Case UC-22: View public conference announcements

**Goal in Context**: A guest views public announcements so they can stay informed about important conference updates.  
**Scope**: Conference Management System (CMS)  
**Level**: User Goal  
**Primary Actor**: Guest  
**Secondary Actors**: Database (announcements), Authorization Service (public access control)  
**Trigger**: The guest navigates to the announcements section.

## Success End Condition
* The guest can view a list of public announcements and read their contents.

## Failed End Condition
* The guest cannot view announcements or only partial information is available, and the system informs the guest of the reason.

## Preconditions
* At least one public announcement exists in the system.
* Announcements are marked as publicly visible.

## Main Success Scenario
1. The guest navigates to the **Announcements** page.
2. The system retrieves public announcements from the database.
3. The system displays a list of announcements ordered by date.
4. The guest selects an announcement to read.
5. The system displays the full announcement content.
6. The guest may return to the list and read other announcements.

## Extensions
* **1a**: No public announcements exist  
  * **1a1**: The system displays a message indicating no announcements are available.
* **2a**: System cannot retrieve announcements due to database error  
  * **2a1**: The system displays a system error message and advises retry later.
* **4a**: Selected announcement is no longer available  
  * **4a1**: The system displays an error and returns the guest to the announcements list.

## Related Information
* **Priority**: Medium  
* **Frequency**: Frequent before and during the conference  
* **Open Issues**:
  * The SRS does not specify announcement categories or expiration behavior.
